# 📊 PitStop Timing Parsing

> Программа для парсинга данных с сайта тайминга заездов и предоставления доступа к ним через REST API и CLI.

**Статус:** в разработке  
**LLM:** Qwen3‑Coder
**Стек:** Django + DRF + PostgreSQL + CLI (Typer + Rich) + BeautifulSoup + Docker

---

# 📚 О проекте

Данный проект предназначен для:

- парсинга данных с сайта: https://timing.batyrshin.name/
- сохранения данных в базу PostgreSQL
- предоставления доступа к данным через REST API
- взаимодействия с системой через CLI‑утилиту
- анализа данных с помощью локальной LLM

Проект разрабатывается как CLI‑инструмент, удовлетворяющий требованиям:

- практическая полезность
- средняя / высокая сложность
- срок реализации: **4–6 недель**
- минимум **7 CLI команд / подкоманд**
- обязательное использование **REST API**

# 🧠 Вывод

Основная сложность проекта:

Разработка на QWEN требует написания качественных подробных промптов, часто могут возникать проблемы с забыванием ИИ 
старого контекста или старого кода. Чем дольше находишься в одном чате, тем дольше поступают ответы. Если первая итерация
время ответа 10-12 секунд, то перед тем как чат зависнет, скорость ответа дошла до 7 минут.

ИИ несет за собой абсолютно весь контекст, из-за этого происходит сильная нагрузка и нежелательно дебагать в том же чате,
так как в таком случае случаются ситуации, при которых ИИ начинает ссылаться совсем не на то. Часто возникают
проблемы с импортами, указываются неверные названия.

Некоторые не очень популярные библиотеки часто неправильно пишутся, в связи с чем приходится дорабатывать.

Нету сильной удовлетворенности в чистоте кода. DRF написан не по принципу "тонких вью", иногда нарушается
принцип единственной ответственности.

Много лапшекода, но подтверждается работоспособность. Несколько раз были применены ИИ устаревшие практики, из-за которых были потенциальные
угрозы безопасности.

Все еще не рекомендуется писать полагаясь только на ИИ. Он пытаестя применять базовую безопасность ввиде CSRF, однако в
коде нету работы с правами. Благо серьезного взаимодействия с правами утилита не требует, так как в ней нету личных данных пользователей.
Однако не применены практики контроля сессии ни при помощи Cookie, ни с помощью jwt.

Общий эффект от QWEN

1) Качество архитектуры 5/5
2) Качество написания кода 3/5
3) Умения поддерживать долгую беседу 1/5 (В сравнении с deepseek, где просто заканчивается место, здесь просто в один момент тормозить начинает)

Некоторые инструменты ИИ записывает для галочки, а потом не применяет (drf-spectacular, ruff)

Очень хорошо проверять линтером весь код, так как видны практики, не соответствующие PEP-8.

Сам ruff не совсем качественно был настроен, так как ругается на Русский язык (не добавлено исключение D415 в .toml)
